# Concept Note: ལས་འགུལ་360

## Project title

**ལས་འགུལ་360 — Project Management System**

## Project overview

**ལས་འགུལ་360** is a responsive full-stack project management web application designed to help individuals, teams, and organizations manage projects, tasks, documents, deadlines, and progress reporting through one centralized digital platform. The first version will focus on a simple, free, and practical web-based system that can later be expanded into a mobile application with advanced features such as budget tracking, approval workflows, field reporting, and AI-assisted report generation.

## Problem statement

Many project activities are still managed through scattered tools such as Excel files, email threads, paper notes, messaging apps, and separate document folders. This creates difficulty in tracking responsibilities, deadlines, task progress, file versions, and overall project status. As a result, project managers and team members often lack a single reliable place to monitor what has been planned, what is pending, what is delayed, and what has been completed.

## Project goal

The goal of **ལས་འགུལ་360** is to develop a simple and user-friendly project management platform that supports project planning, task tracking, document management, and progress monitoring in one integrated dashboard.

## Key objectives

1. To create a centralized digital platform for managing projects and tasks.
2. To enable users to create projects, assign tasks, set deadlines, and track progress.
3. To support task-level comments and file uploads for improved documentation.
4. To provide a dashboard showing project status, pending tasks, completed work, and overdue activities.
5. To design a scalable foundation that can later support mobile access, budget tracking, approval workflows, and field reporting.

## Target users

The initial version will target individual users, small teams, project managers, government staff, research teams, and field-based professionals who need a simple system for organizing and tracking work. In future versions, the system can be customized for forestry, conservation, climate research, public administration, donor-funded projects, and field monitoring activities.

## Version 0 scope

Version 0 will include the minimum essential features required for a working project management system. These include user registration and login, dashboard, project creation, project listing, project detail view, task creation, task status update, task comments, file uploads, and user profile management. Advanced features such as mobile application, AI tools, payment system, advanced roles, Gantt chart, budget tracking, approval workflow, and offline mode will be excluded from the first version.

## Core features

- User registration and login
- Dashboard with project and task summary
- Project creation, editing, and deletion
- Task creation and task status tracking
- Task priority and due date management
- Comments under each task
- File upload for task-related documents
- Basic profile page
- Responsive design for desktop and mobile browser use

## Proposed technology stack

The system will use a free and practical technology stack suitable for a portfolio-grade full-stack application. The proposed stack includes **Next.js** for the web application, **TypeScript** for type-safe development, **Tailwind CSS** and **shadcn/ui** for interface design, **Supabase** for authentication, PostgreSQL database, file storage, and backend services, **GitHub** for version control, and **Vercel** for free web deployment.

## Expected outputs

The main output will be a functional responsive web application named **ལས་འགུལ་360**. The application will allow users to create and manage projects, add and update tasks, upload documents, add comments, and view progress through a dashboard. The project will also include a public GitHub repository, a deployed web link, and basic documentation explaining the system design, features, database structure, and deployment process.

## Future expansion

Future versions may include a dedicated mobile application, GPS-enabled field reporting, budget and expenditure tracking, approval workflows, role-based access control, notifications, Gantt charts, calendar views, AI-generated project summaries, and automated report generation. These features will allow **ལས་འགུལ་360** to evolve from a basic project tracker into a complete project management platform for professional and institutional use.

## Conclusion

**ལས་འགུལ་360** will provide a simple, practical, and scalable foundation for digital project management. By starting with a minimum free version, the system can be developed step by step while maintaining room for future expansion into mobile, AI-assisted, and field-based project management workflows.
